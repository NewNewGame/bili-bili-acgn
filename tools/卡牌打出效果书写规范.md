# 卡牌打出效果书写规范（OnPlay / *Cmd）

面向模组卡牌 `OnPlay`：本质是**按顺序 `await` 串联**「动画 + 各类 `*Cmd`」。攻击管线以 **`DamageCmd.Attack`** 为主；直伤、格挡、能力、牌堆与抽牌、选牌、资源等分别对应下述 API。生成器或手写代码时请保持链式调用风格一致（`.FromCard(this)` → 目标选择 → `.Execute(choiceContext)` 等）。

---

## 1. 攻击（Damage）

**主路径：`DamageCmd.Attack(...)`**

- 最常用链式：`.FromCard(this)` / `.FromOsty(...)`，再  
  `.Targeting(cardPlay.Target)`、`.TargetingAllOpponents(base.CombatState)`、`.TargetingRandomOpponents(...)`，以及  
  `.WithHitCount(n)`、`.Execute(choiceContext)` 等。
- 伤害数值多用 **`base.DynamicVars.Damage.BaseValue`**，或 **`CalculatedDamageVar`**（例如 `Attack(base.DynamicVars.CalculatedDamage)`）。

**不走 Attack 命令的直伤：`CreatureCmd.Damage(...)`**

- 用于自伤、按 `DamageVar` 结算等**不经过 AttackCommand** 的伤害；出现频率低于 `DamageCmd`。

**少数牌：`Hook.ModifyDamage(...)`**

- 在已有逻辑外做额外修正（例如历史记录、条件分支等特殊处理）。

---

## 2. 格挡（Block）

- **`CreatureCmd.GainBlock(creature, base.DynamicVars.Block, cardPlay)`**（或带 `BlockVar` 的重载）。
- 防御牌、伤害+格挡混合牌中很常见。

---

## 3. 能力相关（Power）

- **`PowerCmd.Apply<TPower>(targetCreature, amount, applier, cardSource)`**
- 用于力量/敏捷/虚弱/中毒/各类 Form 能力等；一张牌可连续 `Apply` 多种 Power。

---

## 4. 牌堆与抽牌

| API | 用途 |
|-----|------|
| `CardPileCmd.Draw(choiceContext, count, player)` | 抽牌 |
| `CardPileCmd.Add(card, PileType.Hand / Draw / ...)` | 将指定牌放入某一堆 |
| `CardPileCmd.AddGeneratedCardToCombat` / `AddGeneratedCardsToCombat` | 生成并加入战斗（token、衍生牌等） |

---

## 5. 选牌 UI

- **`CardSelectCmd.FromHand(...)`**
- **`CardSelectCmd.FromSimpleGrid(...)`**

从手牌或列表中选一张或多张，再配合 `CardPileCmd.Add` 或转化逻辑。

---

## 6. 消耗、弃牌与其它单牌操作(CardCmd)

### 6.1 消耗（Exhaust）

```csharp
public static async Task Exhaust(
    PlayerChoiceContext choiceContext,
    CardModel card,
    bool causedByEthereal = false,
    bool skipVisuals = false)
```

- 战斗未结束时：将牌 **`CardPileCmd.Add`** 至 **`PileType.Exhaust`**（默认底部等逻辑以实现为准），写 **`CombatManager.Instance.History`**，并 **`await Hook.AfterCardExhausted(...)`**。
- 若 **`CombatManager.Instance.IsOverOrEnding`** 为真，则不再执行上述流程。

### 6.2 丢弃（Discard）

```csharp
public static async Task Discard(PlayerChoiceContext choiceContext, CardModel card);

public static async Task Discard(PlayerChoiceContext choiceContext, IEnumerable<CardModel> cards);

public static async Task DiscardAndDraw(
    PlayerChoiceContext choiceContext,
    IEnumerable<CardModel> cardsToDiscard,
    int cardsToDraw);
```

- **单张**：`Discard(choiceContext, card)` 内部转为单元素集合再进 **`DiscardAndDraw(..., cardsToDiscard, 0)`**。
- **多张**：`Discard(choiceContext, cards)` 同样走 **`DiscardAndDraw(..., 0)`**。
- **`DiscardAndDraw`**：将牌 **`CardPileCmd.Add`** 到弃牌堆，配合 **`Hook.AfterCardDiscarded`** 等；若 `cardsToDraw > 0` 会在弃牌后继续抽牌；存在 **Sly** 等关键词时还可能再走自动打出等逻辑（以游戏实现为准）。

### 6.3 其它

- **`CardCmd.TransformTo<T>(...)`**
- **`CardCmd.AutoPlay(...)`**
- **`CardCmd.ApplyKeyword(...)`、`CardCmd.Upgrade(...)`**

## 7. 玩家资源与回合

- **`PlayerCmd.GainEnergy` / `LoseEnergy`**
- **`PlayerCmd.GainStars`**（部分牌）
- **`PlayerCmd.EndTurn`**（例如强制结束回合类效果）

---

## 8. 机器人-能力球

- **`OrbCmd.Channel<...>(choiceContext, player)`** 等 — 充能球相关。

---

## 9. 锻造（Forge）

- **`ForgeCmd.Forge(...)`**
- **`ForgeCmd.PlayCombatRoomForgeVfx(...)`**（带表现）

---

## 10. 画面效果相关

- **`CreatureCmd.TriggerAnim(..., "Cast"/"Attack", delay)`** — 施法/攻击动画。
- **`VfxCmd.PlayOnCreatureCenter`、`SfxCmd.Play`**


---

## 11. 其他（零散但会出现）

- **`CreatureCmd.Stun`** — 眩晕。
- **`PotionCmd.TryToProcure`** — 与药水相关（如炼金类）。
- **`CombatManager.Instance.History`** — 读战斗记录做特殊结算（相对较少）。
- **`Cmd.Wait`** — 纯等待。
- 少数牌用 **`TaskHelper.RunSafely(...)`** 包一层异步表现。

---

## 12. 与编辑器的关系

- 编辑器根据 `CardPlayAction` 生成片段时，**伤害类**宜对齐 **`DamageCmd.Attack`** 与目标类型（单体 / 全体 / 随机等）；动态变量名需与卡牌 `DynamicVars` 一致。
- 本文档描述的是**游戏内完整能力面**；生成代码若仅覆盖 Damage / Block / DrawCards 等子集，扩展新类型时请按上表选用对应 `*Cmd` 并保持 `await` 顺序与 `choiceContext` 传递一致。
